## 🎯 Why this exists

Half the web assumes context most readers don't have. Legal contracts, academic papers, dense news analyses, Stack Overflow answers — all written for people who already know the topic.

Three things you might do today:

- **Copy → switch to ChatGPT → paste → wait → switch back.** Lost the page. Lost the flow.
- **Pay $9–25 / month** for an AI sidebar (Monica, Sider, MaxAI) that takes over your screen.
- **Skip the hard sentence** and hope it wasn't important.

ExplainIt is a fourth option: **select a sentence, click once, read the explanation right where you read.** In 13 languages. Free, with Groq.

## 🛠 How it works

1. **Highlight** any text on any webpage
2. **Click** the floating ExplainIt bubble that appears next to your selection
3. **Read** the explanation in a popup, right there in context — copy, retry, or change tone with one click

No tabs to switch. No paste. No interrupting your reading flow.

## ⏱ Get started in 30 seconds

1. **Install** the extension from the [Chrome Web Store](https://chromewebstore.google.com/detail/explainit/hpohkpfddcgchmapdnncddilkgiobabb)
2. **Click the toolbar icon.** The popup will show *"One key, then you're explaining"* with a `Get free key →` button
3. **Open Groq** (the button does it for you), sign up (no credit card), copy your API key
4. **Paste** the key in ExplainIt's Settings, click *Test key*, then *Save*
5. **Select text on any page.** That's it.

English and Russian use hand-tuned native prompt templates. The other 11 reach the model via an English meta-prompt with a target-language steer (*"Output ONLY in X"*) — the same pattern most production multilingual apps use. Arabic output renders right-to-left automatically.

If you hit a language where the output quality feels off, [open an issue](https://github.com/Dimagious/ExplainIt/issues) — we'd rather migrate that language to a native template than ship something shaky.

## 🔒 Privacy

- Your **API key never leaves your device.** It lives only in `chrome.storage.local`, which is never synced to Google or anywhere else.
- The **selected text** you highlight is sent **only** to the provider you picked. Nothing routes through any ExplainIt server, because there is no ExplainIt server.
- **No browsing history collection.** No analytics. No telemetry. No signup. No email. No account.
- The full policy lives at [`privacy-policy.html`](privacy-policy.html) (also hosted on GitHub Pages — linked from the Chrome Web Store listing).

## ❓ FAQ

<details>
<summary><b>Is my API key safe?</b></summary>

<br/>

Yes. The key is stored only in `chrome.storage.local` on your device. It is never synced via Google Sync, never sent to any ExplainIt server (there isn't one), and never logged. When you explain text, the extension sends the request directly from your browser to the AI provider you chose — your key in an `Authorization` / `x-api-key` / `x-goog-api-key` header, nothing else.

</details>

<details>
<summary><b>How much does it cost?</b></summary>

<br/>

**Free, if you use Groq** — the recommended default. No credit card, no subscription.

If you switch to a paid provider (OpenAI / Gemini / Anthropic), you pay that provider directly per request — roughly **$0.0001 – $0.0003 per explanation** depending on the model. For a typical reader that's cents per month.

There is no ExplainIt subscription. There never will be one.

</details>

<details>
<summary><b>Does it work offline?</b></summary>

<br/>

No. ExplainIt sends the selected text to your chosen AI provider's API, which requires an internet connection. The extension itself does not include an offline language model — that would balloon the package size and degrade output quality.

</details>

<details>
<summary><b>Which browsers are supported?</b></summary>

<br/>

Chromium-based browsers that accept Chrome Web Store extensions: **Chrome, Edge, Brave, Arc, Opera, Vivaldi**. Manifest V3 compatible.

Firefox and Safari are not currently supported — they require separately packaged extensions with different APIs. If there's demand, open an issue.

</details>

<details>
<summary><b>Does the AI provider train on my selected text?</b></summary>

<br/>

ExplainIt does not train on anything — it has no servers and no data store. Whether the underlying provider trains on API requests is determined by **that provider's terms**:

- **Groq:** does not train on API inputs by default (per their privacy policy as of May 2026).
- **OpenAI:** API data is not used for training by default (Enterprise / standard API terms).
- **Anthropic:** API data is not used for training by default.
- **Google Gemini:** AI Studio free tier does train; paid Gemini API does not.

Check your chosen provider's current terms — we link to each from the Settings panel.

</details>

<details>
<summary><b>Can I add a language?</b></summary>

<br/>

Yes — see [Contributing](#-contributing). The TL;DR: add a language code to `LANGUAGE_NAMES` in `extension/config.js`, add a `<option>` to the Settings dropdown, and (optionally) write a native prompt template if you want highest-quality output instead of the English meta-prompt path. PR welcome.

</details>

<details>
<summary><b>The Arabic output looks wrong — how do I report it?</b></summary>

<br/>

[Open an issue](https://github.com/Dimagious/ExplainIt/issues) with:

1. The English source sentence you selected
2. The Arabic output you got
3. What you expected (rough native-speaker fix is fine)
4. The provider you used (Groq, OpenAI, etc.)

We'll either tune the meta-prompt steer for that script, migrate the language to a native template, or both. Same flow applies to Hindi, Vietnamese, Turkish, or any other language.

</details>

<details>
<summary><b>Why do I have to grant permission to api.openai.com / api.groq.com / etc.?</b></summary>

<br/>

The extension can't contact the AI provider's API without that permission — Chrome's security model requires explicit host access for any cross-origin fetch.

Since v2.0.2, host access is requested **at runtime** (when you pick a provider and click *Test key*), not at install time. You only ever grant access to the one provider you actually use. The install dialog itself stays short.

</details>

- 🌟 **Star the repo** if ExplainIt has been useful to you — it nudges the GitHub algorithm and brings in contributors
- 🐛 **Report a bug** by [opening an issue](https://github.com/Dimagious/ExplainIt/issues/new) — include the provider, language, tone, and selected text if relevant
- 🌍 **Improve a language** — if you're a native speaker of Hindi, Arabic, Vietnamese, Turkish, or any of the meta-prompt languages, we'd love your eyes on output quality

### Code contribution flow

1. Open an issue first for anything bigger than a typo — saves wasted PR work if direction is off
2. Fork, branch, make the change with a test (see [`extension/tests/`](extension/tests/) for the patterns)
3. Run `npm test` in `extension/` — must stay green
4. Open a PR; link the issue; explain *why* in the body

## 👩‍💻 For developers

<details>
<summary><b>Architecture, local setup, testing, file map</b></summary>

<br/>

### Architecture

```text
Page Selection
   ↓
Content Script + Inline Popup UI
   ↓
Background Service Worker
   ↓
Selected AI Provider API (Groq / Gemini / OpenAI / Anthropic)
```

No dedicated server is required for extension runtime. Provider hosts are requested **at runtime** (when the user picks a provider and clicks *Test key*), not at install time.

### Local setup

```bash
git clone https://github.com/Dimagious/ExplainIt.git
cd ExplainIt/extension
npm ci
```

Then in Chrome: `chrome://extensions/` → enable *Developer mode* → *Load unpacked* → select the `extension/` folder.

### Run tests

```bash
cd extension
npm test
```

138 Jest tests covering provider call shapes, retry logic, error classification, multi-language meta-prompt path, key-prefix validation, storage routing, and the setupCompleted flag.

### Storage model

`chrome.storage.local` (per-install, never synced):

- `provider` — chosen provider id
- `apiKeys` — per-provider key map: `{ groq: '...', openai: '...', ... }`
- `setupCompleted` — sticky boolean, true after first successful save with a key

`chrome.storage.sync` (cross-device preferences):

- `language` · `tone`

### Manifest permissions

- `permissions: ["storage"]` — silent at install time
- `optional_host_permissions` for the 4 provider APIs — requested at runtime when the user picks a provider, NOT at install
- Content scripts on `<all_urls>` — required to detect text selection on any page

### Key files

- [`extension/manifest.json`](extension/manifest.json) — MV3 manifest
- [`extension/popup.html`](extension/popup.html), [`popup.js`](extension/popup.js), [`popup.css`](extension/popup.css) — toolbar popup
- [`extension/background.js`](extension/background.js) — service worker, provider dispatch, `classifyError` for typed UI copy, `buildPrompt` with meta-prompt fallback
- [`extension/content.js`](extension/content.js), [`inline-popup.js`](extension/inline-popup.js) — in-page floating bubble + inline settings
- [`extension/config.js`](extension/config.js) — provider metadata, native prompt templates (EN/RU), `META_PROMPTS`, `LANGUAGE_NAMES`
- [`extension/tests/`](extension/tests/) — Jest unit tests

### Repository structure

```
extension/                 — active product code
extension/tests/           — Jest unit tests
extension/icons/source/    — editable SVG masters for the toolbar icon
store-assets/              — Chrome Web Store text, shot list, promo brief, dashboard checklist
docs/marketing/            — README assets (screenshots, hero animation source)
privacy-policy.html        — public privacy policy
```

### Adding a new language

To add a new output language, e.g. Italian (`it`):

1. Add `it: 'Italian'` to `LANGUAGE_NAMES` in [`config.js`](extension/config.js) **and** the mirror in [`background.js`](extension/background.js)
2. Add `it` to the `validLanguages` array in `validateSettings` ([`popup.js`](extension/popup.js))
3. Add `it: '🇮🇹 Italiano'` to `STATUS_LANGUAGE_LABELS` in [`popup.js`](extension/popup.js) and `INLINE_LANG_LABELS` in [`inline-popup.js`](extension/inline-popup.js)
4. Add an `<option value="it">🇮🇹 Italiano</option>` to both the toolbar Settings dropdown ([`popup.html`](extension/popup.html)) and the inline Settings dropdown ([`inline-popup.js`](extension/inline-popup.js))
5. Add a smoke test in [`config.test.js`](extension/tests/config.test.js)
6. **Optional, higher quality:** add a native prompt template under `PROMPTS` in both `config.js` and `background.js` — this skips the meta-prompt fallback for that language

</details>
